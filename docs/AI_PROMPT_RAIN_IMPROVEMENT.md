# 🤖 ПРОМПТ ДЛЯ ИИ: УЛУЧШЕНИЕ ВИЗУАЛИЗАЦИИ ДОЖДЯ В THREE.JS

## 📋 Контекст проекта

У меня есть React приложение погоды с Three.js, использующее два компонента дождя:

1. **RealisticRainStreaks** - реалистичный дождь с физически корректными материалами
2. **ImprovedRainStreaks** - упрощённый дождь для производительности

**Проблема:** В ImprovedRainStreaks капли слишком длинные (0.6-1.3 единиц), выглядят как палки вместо капель.

---

## 🎯 Задача для ИИ

Создай **оптимизированный компонент дождя для Three.js + React**, который:

### Обязательные требования:

1. **Правильная длина капель:**
   - Минимум: 0.18 единиц
   - Максимум: 0.28 единиц
   - Формула: `0.18 + Math.random() * 0.1`

2. **Производительность:**
   - Использовать InstancedMesh
   - Поддержка 800-1000 капель при 60 FPS
   - Подходит для mobile устройств

3. **Визуальное качество:**
   - Полупрозрачные капли
   - Плавная анимация падения
   - Покачивание от ветра
   - Система брызг при приземлении

4. **Настраиваемость:**
   - Props для количества капель
   - Контроль интенсивности
   - Скорость и направление ветра
   - Включение/выключение брызг

### Физика падения:
```javascript
// Каждая капля должна иметь:
{
  x, y, z,              // Позиция
  speed: 15-25,         // Скорость падения
  length: 0.18-0.28,    // Длина капли (ВАЖНО!)
  thickness: 0.012-0.02, // Толщина
  rotation,             // Вращение
  offset,               // Временной сдвиг для покачивания
}

// Анимация:
- Падение вниз со скоростью speed * intensity
- Покачивание: Math.sin(time * 2 + offset) * 0.02
- Влияние ветра: windSpeed * Math.cos(windDirection)
- При достижении y < 0: создать брызги и сбросить наверх
```
