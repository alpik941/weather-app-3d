## Обнаруженные и исправленные баги

### 🐛 Баг #1: Отсутствие сброса флагов оффлайн режима в handleFindMe
**Проблема**: При использовании кнопки "Find My Location" флаги `offlineUsed` и `offlineTs` не сбрасывались в начале функции, что могло приводить к отображению баннера оффлайн режима даже при успешной загрузке данных.

**Исправление**: Добавлены строки:
```javascript
setOfflineUsed(false);
setOfflineTs(null);
```
в начало функции `handleFindMe` после `setLoading(true)` и `setError(null)`.

**Файл**: [src/App.jsx](src/App.jsx#L140)

---

### 🐛 Баг #2: Неправильное форматирование времени в баннере оффлайн
**Проблема**: В отображении времени в оффлайн баннере был пробел в выражении `offlineTs/1000` вместо `offlineTs / 1000`, что могло вызывать проблемы с читаемостью кода.

**Исправление**: Добавлены пробелы вокруг оператора деления для соответствия стандартам форматирования:
```javascript
{formatDate(Math.floor(offlineTs / 1000), { opts: { month: 'short', day: 'numeric' } })}
{formatTime(Math.floor(offlineTs / 1000))}
```

**Файл**: [src/App.jsx](src/App.jsx#L467)

---

### 🐛 Баг #3: Не восстанавливается timezone при загрузке из кеша
**Проблема**: При загрузке данных из кеша в оффлайн режиме не устанавливался timezone, что приводило к неправильному отображению времени в прогнозах.

**Исправление**: Добавлена установка timezone из кешированных данных в двух местах:

1. В `fetchWeatherData` (fallback по названию города):
```javascript
// Restore timezone from cached weather data
try {
  const tzCandidate = weather?.sys?.tz_id ?? weather?.timezone;
  if (tzCandidate !== undefined && tzCandidate !== null) setTimezone(tzCandidate);
} catch {}
```

2. В `handleFindMe` (fallback по координатам):
```javascript
// Restore timezone from cached weather data
try {
  const tzCandidate = p.weather?.sys?.tz_id ?? p.weather?.timezone;
  if (tzCandidate !== undefined && tzCandidate !== null) setTimezone(tzCandidate);
} catch {}
```

**Файлы**: 
- [src/App.jsx](src/App.jsx#L119) (fetchWeatherData)
- [src/App.jsx](src/App.jsx#L194) (handleFindMe)

## Проверенная функциональность

### ✅ Кеширование данных
- **Статус**: Работает корректно
- **Проверка**: Данные сохраняются в localStorage при каждом успешном запросе
- **Ключи кеша**:
  - По названию города: `weather-cache:v1:bundle:<city>`
  - По координатам: `weather-cache:v1:bundle:coords:<lat>,<lon>`

### ✅ Загрузка из кеша при отсутствии сети
- **Статус**: Работает корректно
- **Проверка**: При ошибке сети приложение автоматически загружает данные из кеша
- **Сценарии**:
  1. Поиск города по имени
  2. Определение местоположения (Find My Location)
  3. Повторный запрос ранее загруженного города

### ✅ Отображение баннера оффлайн режима
- **Статус**: Работает корректно
- **Проверка**: Баннер отображается только при использовании кешированных данных
- **Визуал**:
  - Светлая тема: желтый фон `bg-yellow-100/70`, темный текст `text-yellow-800`
  - Темная тема: полупрозрачный желтый фон `bg-yellow-900/30`, светлый текст `text-yellow-200`

### ✅ Отображение timestamp последнего обновления
- **Статус**: Работает корректно
- **Проверка**: В баннере показывается дата и время последней успешной загрузки
- **Формат**: "Jan 11, 3:45 PM" (в зависимости от локали и настроек времени)

### ✅ Многоязычность
- **Статус**: Работает корректно
- **Проверка**: Баннер переведен на 10 языков
- **Языки**: EN, ES, FR, DE, IT, PT, RU, ZH, JA, KO

### ✅ Восстановление всех данных из кеша
- **Статус**: Работает корректно
- **Проверка**: При загрузке из кеша восстанавливаются:
  - Текущая погода (weather)
  - 5-дневный прогноз (forecast)
  - Почасовой прогноз на 3 дня (hourly)
  - Недельный прогноз (weekly)
  - Погодные предупреждения (alerts)
  - Рекомендации по активностям (activities)
  - Качество воздуха (airQuality)
  - Timezone

### ✅ Обработка ошибок
- **Статус**: Работает корректно
- **Проверка**: При отсутствии кеша показывается сообщение об ошибке
- **Сообщение**: "Failed to load weather data." (переведено на все языки)

## Тестирование

### Как протестировать вручную

1. **Подготовка**:
   ```bash
   npm run dev
   ```
   Откройте http://localhost:5174 в браузере

2. **Тест оффлайн режима**:
   - Откройте DevTools (F12) → Application → Local Storage
   - Дождитесь загрузки данных для London
   - Проверьте наличие ключа `weather-cache:v1:bundle:london`
   - DevTools → Network → выберите "Offline"
   - Перезагрузите страницу (F5)
   - **Ожидаемый результат**: Данные отображаются, виден желтый баннер "Offline snapshot" с датой/временем

3. **Тест поиска города оффлайн**:
   - Онлайн: найдите Paris, дождитесь загрузки
   - Онлайн: найдите Tokyo, дождитесь загрузки
   - Включите режим Offline
   - Найдите Paris через поиск
   - **Ожидаемый результат**: Данные Paris загружаются из кеша, показан баннер

4. **Тест Find My Location оффлайн**:
   - Онлайн: нажмите кнопку с иконкой Locate
   - Разрешите доступ к геолокации
   - Дождитесь загрузки данных
   - Включите режим Offline
   - Нажмите Locate снова
   - **Ожидаемый результат**: Данные загружаются из кеша, показан баннер
